Class com.xerox.integrityperformancetool.IntegrityStatisticBean

java.lang.Object

A java bean abstracting a single statistic retrieved from a Server Statistics .csv file. Here, a "statistic" is representative of a single line in the Server Statistics .csv file.

NOTE: The Server Statistics .csv file can be generated from an Integrity Server via the Integrity Admin Client or the CLI command "integrity stats --csv > csvFileName.csv".

An IntegrityStatisticBean object has the following private properties:

  • average (java.lang.Long) - The average value of the statistic. Correlates with the statistic's "Average" column value in the .csv file. The default value is "0"
  • count (java.lang.Long) - The count of the number of times the statistic was measured within the time period between the "Start Date" and "End Date" of the statistic. Correlates with the statistic's "Count" column value in the .csv file. The default value is "0"
  • endDate  ( java.util.Calendar) - A Calendar object set to the value of the End Date of the period which the statistic was captured within. Correlates with the statistic's "End Date" column value in the .csv file. The default value is a Calendar object set to the date and time which the PerformanceTool class was executed (same as the PerformanceTool's runTime property)
  • group (java.lang.String)- The Group of the statistic used to identify the grouping of statistics that this statistic belongs to. Correlates with the "Group" column value in the .csv file. The default value is "None".
  • kind (java.lang.String) - The Kind of statistic used to identify the statistic. Correlates to the "Kind" column of the .csv file. The default value is "None".
  • max (java.lang.Long) - The maximum value of the statistic that was captured between the Start Date and End Date time period. Correlates with the "Maximum" column value in the .csv file. The default value is "0".
  • min (java.lang.Long) - The minimum value of the statistic that was captured between the Start Date and End Date time period. Correlates with the "Minimum" column value in the .csv file. The default value is "0"
  • mode (java.lang.String) - The mode of the statistic captured, either "delta" or "cumulative". Correlates to the "Mode" column value in the .csv file. The default value is "none".
  • name (java.lang.String) - The main identifier of the statistic. Correlates with the "Statistic" column value in the .csv file. The default value is "None".
  • startDate (java.lang.Calendar) - A Calendar object set to the value of the Start Date of the period which the statistic was captured within. Correlates with the "Start Date" column value in the .csv file. The default value is a Calendar object set to the date and time which the PerformanceTool class was executed (same as the PerformanceTool's runTime property)
  • sum ( ?? ) - TBD
  • totalCount (java.lang.Long)- the cumulative value of the "Count" value, recorded since the installation of the Integrity Server. Correlates with the "Total Count" column value in the .csv file. The default value is "0".
  • unit (java.lang.String) - The unit of measurement for the statistic. Correlates with the "Unit" column value in the .csv file.The default value is "None".

Constructor Summary
IntegrityStatisticBean()
Creates an IntegrityStatisticBean object with all property values set to null. Due to the large number of available properties, no other variations of the Constructor are provided. It is recommended a default IntegrityStatisticBean is created, and then the properties be manipulated via the get/set methods available.
